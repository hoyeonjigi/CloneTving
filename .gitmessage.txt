# Chore : 코드의 수정 없이 설정을 변경
# Fix : 오류 및 코드 수정
# Style : 코드 포맷팅, 코드 변경이 없는 경우 및 세미콜론 누락
# Docs : 리드미 및 문서 변경
# Comment : 필요한 주석 추가 및 변경
# Refactor : 코드 리펙토링
# Rename : 파일 혹은 폴더명을 수정하거나 옮기는 작업만 수행한 경우
# Test : 테스트 코드, 리펙토링 테스트 코드 추가
# !HOTFIX : 급하게 치명적인 버그를 처리해야하는 경우
